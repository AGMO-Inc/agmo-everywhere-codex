# Agmo Design

Treat this as a compatibility alias of `brainstorming`, not as a separate workflow stage.

## Main behavior

1. route to the `brainstorming` workflow semantics
2. clarify the desired outcome and decision boundaries
3. compare candidate approaches with explicit tradeoffs
4. recommend one direction while keeping coding blocked
5. transition to `plan` only after the direction is accepted

## Canonical name

Use `brainstorming` in docs, onboarding, and workflow descriptions. Keep `design` only so existing prompts and habits still route correctly.
