# Agmo Design

Treat this as a compatibility alias of `brainstorming`, not as a separate workflow stage.

## Main behavior

1. route to the `brainstorming` workflow semantics
2. clarify the outcome and boundaries
3. compare candidate approaches
4. recommend one direction with tradeoffs
5. keep coding blocked until the design is accepted

## Canonical name

Use `brainstorming` in docs, onboarding, and workflow descriptions. Keep `design` only so existing prompts keep routing correctly.
