# Agmo Brainstorming

Use this when the direction is still fluid and implementation should stay blocked.

This is the canonical first-stage workflow name. Requests phrased as `design` should route here as compatibility alias behavior rather than as a separate stage.

## Main behavior

1. inspect local context before asking broad repo questions
2. use `agmo-planner` as the primary design-thinking lane
3. compare 2-3 approaches with tradeoffs
4. avoid starting implementation until the direction is accepted

## Typical next step

- accepted direction -> `plan`
- still too ambiguous -> keep brainstorming
